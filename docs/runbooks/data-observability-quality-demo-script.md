# 数据可观测性与质量平台演示剧本

本文是一份可直接照着讲的演示脚本，面向规则口径、异常信号、告警路径和修复闭环场景。

## 1. 演示目标

- 说明规则命中不等于告警有效
- 说明 `/tdd` 如何前置锁定误报 / 漏报和处置责任
- 说明 `/verify` 如何把告警结果转成可 review 的结论

## 2. 演示脚本

### Step 1. 先讲规则与告警不是一个层次

```text
数据质量平台最容易被误判的是：规则命中了，不代表告警就可行动；没有误报、漏报和责任路径，平台价值不完整。
```

### Step 2. 用 `/team-intake` 锁边界

```text
/team-intake
目标：为数据可观测性平台新增质量规则并补齐告警与修复闭环
范围：规则定义、异常检测、告警通道、结果汇总、测试计划
不做：无关分析台 UI 改造
约束：必须说明规则口径、误报/漏报风险、告警责任和处置路径
```

### Step 3. 用 `/verify` 收口

```text
/verify
请基于当前质量规则改动，输出异常检测结果、误报/漏报、告警有效性和可直接进入 /team-review 的结论。
```
