# Team Intake 完整示例

本文对标 [team-command-output-contracts.md](team-command-output-contracts.md) 和 [../../commands/team-intake.md](../../commands/team-intake.md)，给出可直接参考的完整 intake 输出示例。

## 1. 场景

- 任务：为审批列表新增导出能力，并补权限限制与前端导出按钮
- 涉及：前后端、QA、可能的发布验证

## 2. 输入示例

```text
/team-intake
目标：为审批列表新增导出能力
范围：前端导出入口、后端导出接口、权限限制、测试计划
不做：离线任务平台重构
约束：本周内完成，必须明确是否命中 私有权限中心 权限中心
```

## 3. 完整输出示例

```text
需求简报
- 目标：为审批列表提供可控的导出能力，支持当前筛选条件导出
- 成功标准：具备导出入口、权限受控、导出结果字段正确、关键链路可验证
- 范围外：不重做离线导出平台，不新增复杂模板能力

参与角色清单
- tech-lead
- product-manager
- backend-engineer
- frontend-engineer
- qa-engineer

待确认项列表
- 导出是否同步完成，还是走异步任务
- 权限限制来自现有角色权限还是独立导出权限

企业治理待确认项
- 是否企业内部应用：是
- 应用等级：待确认
- 技术架构等级：待确认
- 数据 / 合规风险：导出包含审批数据，需确认字段脱敏要求
- 集团组件 / 平台约束：疑似命中 私有权限中心 权限中心

领域技能包启用建议
- 候选领域能力：私有权限中心 权限相关 company 能力
- 触发原因：导出权限与现有资源权限相关
- 启用方式：进入 /team-plan 后确认
- 下一步责任角色：tech-lead、architect

UI 范围、终端假设与质量门禁
- UI 范围：审批列表页新增导出按钮与状态反馈
- 终端假设：桌面端优先，iPad 横屏需验证
- 质量门禁：需要响应式验证、权限态验证、空态与错误态说明
```

## 4. 常见错误

- 只写目标，不写范围外事项
- overlay 候选项没有进入输出
- UI 任务没有写质量门禁
