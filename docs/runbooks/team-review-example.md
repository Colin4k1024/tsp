# Team Review 完整示例

本文对标 [team-command-output-contracts.md](team-command-output-contracts.md) 和 [../../commands/team-review.md](../../commands/team-review.md)，给出评审结论、阻塞项和放行建议的完整示例。

## 1. 场景

- QA 已完成导出能力验证
- 需要输出统一评审结论，判断是否可进入发布准备

## 2. 输入示例

```text
/team-review
基于当前导出能力的实现说明、自测结果和 QA 验证结果，输出评审结论、阻塞项和放行建议。
```

## 3. 完整输出示例

```text
评审结论
- 当前版本功能完整，权限主路径和导出结果正确性通过验证
- 允许进入灰度发布准备，但需带观察项

阻塞项
- 无阻塞项

放行建议
- 建议：有条件通过
- 条件：灰度阶段需重点观察导出成功率与错误率

前端质量门禁检查结果
- 视觉一致性：通过
- 交互完整性：通过
- 边界态：通过
- 无障碍与性能：通过

领域扩展约束核对结果
- 能力名：私有权限中心 权限映射能力
- 关键约束：按钮展示与接口权限判断必须一致
- 证据：权限用例验证、页面行为验证、接口响应验证
- 状态：通过
- 残余风险：大数据量导出性能仍需在灰度阶段继续观察

残余风险与后续动作
- 风险：导出量大时的性能波动
- 动作：DevOps 在灰度阶段观察接口耗时和错误率
```

## 4. 常见错误

- 把所有问题都写成阻塞项，无法判断是否可放行
- 没有把前端质量门禁和领域约束分开
- 放行建议没有附带条件和观察项
