# Team Execute 完整示例

本文对标 [team-command-output-contracts.md](team-command-output-contracts.md) 和 [../../commands/team-execute.md](../../commands/team-execute.md)，给出实现、自测和交接准备的完整输出示例。

## 1. 场景

- 延续导出能力需求
- 前后端均已按计划完成本轮实现
- 需要把实现结果、自测结论和领域扩展记录整理给 QA

## 2. 输入示例

```text
/team-execute
按当前 plan 执行审批列表导出能力实现。
请汇总前后端改动、自测结果、交给 QA 的说明，以及 私有权限中心 相关执行记录。
```

## 3. 完整输出示例

```text
实现结果
- 后端新增 /api/approval/export 接口，支持按当前筛选条件导出
- 前端新增导出按钮、loading 状态、失败提示和无权限态隐藏逻辑
- 权限判断接入现有 私有权限中心 资源权限映射

自测结论
- 后端：接口成功、无权限、参数缺失 3 条路径已验证
- 前端：桌面端与 iPad 横屏验证通过，loading / 成功 / 失败态可见
- 联调：前后端字段映射一致，导出文件字段与页面筛选条件匹配

交给 QA 的说明
- 重点验证 3 组权限场景：有权限、无权限、权限变更后重新进入页面
- 重点验证导出失败提示与重复点击保护
- 已知限制：超过 1 万条记录的导出压测未覆盖

领域扩展执行记录
- 能力名：私有权限中心 权限映射能力
- 任务变量：resourceCode=approval.export, scope=approval:list
- 输入来源：现有权限配置、接口契约、需求说明
- 已读取资料：权限映射说明、导出接口契约、列表页交互说明
- 执行结果：完成按钮展示条件和接口权限校验接入
- 回落位置：已归并到实现结果、自测结论与已知限制

必要的 UI 检查清单
- 响应式：桌面端、iPad 横屏通过
- 异常态：失败提示和无权限态通过
- 可访问性：按钮可聚焦，状态变化有文本反馈
```

## 4. 常见错误

- 只写代码改了什么，不写自测覆盖了什么
- company 执行记录存在，但没有回落到主链输出
- 交给 QA 的说明过于笼统，无法直接执行
