# Executive Briefing Talk Track

这份讲稿面向管理层、平台赞助人或需要快速理解平台价值的人。重点不是命令细节，而是为什么这个平台值得继续推广。

## 1. 3 分钟版本

### 第 1 分钟：平台定位

建议讲法：

“Team Skills Platform 不是 prompts 仓库，也不是单个 Agent 技能集合，而是一套把需求、方案、研发、测试、发布串成角色化协作链路的平台。”

### 第 2 分钟：本轮升级重点

建议讲法：

“这一轮升级做了三件真正影响落地的事。第一，公开命令面更完整，`/tdd` 和 `/harness-audit` 让交付闭环和平台治理都能标准化；第二，runtime 具备 memory、observe、budget、compact 等增强能力，长会话可控；第三，已经形成 9 条完整 vertical 演示闭环，平台不再只停留在通用模板。”

### 第 3 分钟：管理价值

建议讲法：

“对管理层来说，最重要的不是某个命令，而是这套平台把角色边界、交接质量和材料复用都沉淀下来了。它能缩短项目接入成本，也能降低平台能力升级后的说明成本。”

## 2. 8 分钟版本

### 第一段：为什么不是普通 AI 配置仓库

- 讲 8 角色矩阵
- 讲主链和 handoff
- 讲 specialist 只是加速器，不是平行责任链

### 第二段：为什么现在比之前更可用

- 讲 `/tdd` 和 `/harness-audit`
- 讲 runtime 增强
- 讲 9 条 vertical 闭环

### 第三段：为什么适合继续推广

- 讲 onboarding 材料已成体系
- 讲 vertical 演示已可按听众选路由
- 讲仓库校验仍然通过，没有破坏基线
